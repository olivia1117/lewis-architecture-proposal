Application Overview

This webpage explains my project for credit in CPSC 61200 - Software Architecture and Design. The main goal is to help Lewis faculty and students teach and understand essential computer science concepts without worrying about switching languages, frameworks, or tools.

The project, called Lewis Architecture, is modular and designed for students at all levels. It begins with basic static web development and progresses through React, cloud deployment, authentication, databases, APIs, and other industry-relevant technologies.

Students follow along with written tutorials provided on the site, which includes example code, documentation, and a complete multi-stage version of the project.

The project is open source and available for the entire Computer Science community.

Link to Architecture Resources: https://www.lewisunivcs.com/

Stages of Learning:
  • Level 0 — Start with the Base Code in React
  • Level 1 — Enhance UX/UI and Learn Basic React + API Calls
  • Level 2 — Azure Hosting + CI/CD
  • Level 3 — Custom Domain + Auth0 Authentication
  • Level 4 — Node.js Server + MongoDB Atlas
  • Level 5 — Server-Side File Storage & Sharing

Technology Overview

Programming Languages

  • JavaScript / TypeScript — Primary front-end languages; TypeScript optional
  • HTML5 / CSS3 — Base for early-stage static development
  • Node.js — Used for API and server development

Frameworks & Libraries

  • React.js — Component-driven front-end UI
  • Vite / Create React App — Project setup tools
  • Auth0 React SDK — Authentication
  • Mongoose — MongoDB object modeling for Node.js

Cloud Platforms

  • GitHub — Version control + CI/CD
  • Azure Static Web Apps — Cloud hosting
  • MongoDB Atlas — Managed database

Authentication & Security

  • Auth0 — OAuth2 / OIDC login
  • .env / GitHub Secrets / Azure Settings — Secret & key management

Domain & Hosting

  • GoDaddy — Domain registration
  • Azure DNS — Domain → Azure SWA routing

File & Object Storage

  • Azure Blob Storage — Upload and store assets
  • Firebase Storage — Optional alternative

Tooling & Environment

  • Visual Studio Code
  • GitHub Actions — Automated builds & deploys

Feature List

  • Basic React Tic-Tac-Toe Game — 3×3 grid, player turns, winner detection, reset
  • Styling & Theming — responsive layout, CSS modules or Tailwind, light/dark toggle
  • Static Hosting on Azure — Azure Static Web Apps deployment
  • Custom Domain Integration — GoDaddy → Azure DNS mapping
  • Authentication with Auth0 — login/logout, show user name/email, protect game route
  • MongoDB Atlas Integration — serverless API to record login events and game results (winner, timestamp, moves)
  • File Storage Integration — upload profile picture to Azure Blob or Firebase Storage, display in UI
  • Protected Match History — authenticated users can view saved games
  • Play Again Flow — quick reset and rematch UX
  • Basic error handling & loading states

Future Feature List

  • Online multiplayer mode — real-time state syncing with WebSockets or Azure SignalR
  • Matchmaking Lobby — create/join rooms to play against others
  • AI Opponent — difficulty levels: random, heuristic, minimax
  • Leaderboard — rank players by wins / win rate
  • Achievement Badges — reward streaks and milestones
  • Replay System — save game JSON for step-by-step playback
  • User Settings Page — change avatar, theme, display name, preferences
  • PWA / Offline Mode — installable app with cached assets
  • Mobile App — React Native port
  • Extended Serverless APIs — analytics, email notifications

Basic Technical Features

  • React component architecture — functional components, props, hooks
  • Game logic & state — win/draw detection, resets, and state separation
  • Routing — react-router for /play, /settings, /history
  • Local persistence — localStorage for theme & local scores
  • Azure Static Web App hosting — deploy front-end to cloud
  • GitHub Actions CI/CD — automated build & deploy pipeline
  • Auth0 integration — secure login and protected routes
  • REST API calls — Axios or Fetch to backend/serverless functions
  • MongoDB Atlas connectivity — save and read game history
  • Environment variable management — .env, GitHub Secrets, Azure app settings

Advanced Technical Features

  • Role-based access control (RBAC) — admin vs. user permissions via Auth0
  • Serverless backend (Azure Functions) — scalable endpoints for game logging and analytics
  • Real-time multiplayer — WebSockets or Azure SignalR
  • Production logging & monitoring — Application Insights or equivalent
  • Database schema design & indexing — optimized collections for game_sessions, user_profiles
  • Caching strategy — reduce DB load (Redis or in-memory)
  • Containerized backend — Docker images, optional Azure Container Apps deployment
  • CI/CD with tests & staging — unit tests, integration tests, preview environments
  • Security hardening — JWT validation, rate limiting, CORS, secret rotation
  • Performance optimization — code-splitting, React.memo, lazy-loading
